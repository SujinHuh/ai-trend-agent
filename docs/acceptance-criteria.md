# Acceptance Criteria

이 문서는 전체 제품 기준의 수용 기준이다.

Task 001처럼 개별 하네스 작업은 각 작업 폴더의 `requirements.md`와 `validation_report.md`를 우선 기준으로 삼는다.

## 1. Daily Report

- 매일 지정된 시간에 리포트 생성 작업이 실행된다.
- 리포트는 최근 24시간 또는 설정된 수집 기간의 항목만 포함한다.
- 각 항목은 제목, 요약, 원문 링크, 출처, 게시일, 검증 상태를 포함한다.
- 원문 링크가 없는 항목은 메인 리포트에 포함하지 않는다.

## 2. Ingestion

- 설정된 Source Registry에 있는 활성 소스만 수집한다.
- 수집 실패한 소스는 실패 원인을 로그에 남긴다.
- 같은 URL은 중복 저장하지 않는다.
- 같은 주제의 중복 항목은 대표 항목 하나로 묶는다.

## 3. Verification

- 게시일이 수집 기간 밖이면 제외하거나 낮은 우선순위로 이동한다.
- 공식 출처가 아닌 항목은 신뢰도 점수를 낮게 부여한다.
- 가격, 요금제, 모델명, 버전 번호 변경은 원문 근거가 없으면 확정 표현을 쓰지 않는다.
- LLM 요약은 원문에 없는 내용을 사실처럼 추가하지 않는다.

## 4. Email

- 이메일 제목에는 리포트 날짜가 포함된다.
- 이메일 상단에는 오늘의 핵심 항목 3-5개가 표시된다.
- 상세 항목에는 원문 링크가 포함된다.
- 발송 실패 시 재시도하고 실패 로그를 남긴다.

## 5. Notion

- 생성된 리포트가 Notion에 날짜별로 저장된다.
- 각 항목은 카테고리, 중요도, 출처, 검증 상태로 필터링할 수 있다.
- Notion 저장 실패 시 이메일 발송은 계속 진행된다.

## 6. TTS

- 음성용 스크립트는 5-10분 분량으로 생성된다.
- 스크립트는 구어체로 작성된다.
- TTS 생성 실패 시 스크립트만 전달한다.
- 오디오 파일이 생성되면 이메일 또는 Notion에서 접근할 수 있다.

## 7. Multi-LLM Review

- 1차 요약 결과는 최소 한 번 이상 다른 검토 단계로 전달된다.
- 중요도 상위 항목은 추가 LLM 리뷰 대상이 된다.
- LLM 리뷰 결과는 통과, 수정 필요, 근거 부족 중 하나로 표시된다.
- 근거 부족 항목은 메인 리포트에서 분리된다.
